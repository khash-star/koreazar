# Workflows

Task playbooks for Cursor. Pick **one** that best matches the request.

## Domain playbooks (repo root of `project-memory/`)

| Playbook | When |
|----------|------|
| `../WORKFLOW_CHAT.md` | Messages, Chat, conversations, admin broadcast |
| `../WORKFLOW_LISTINGS.md` | Listings CRUD, Home, PHP API, saved listings |
| `../WORKFLOW_AUTH.md` | Login, register, phone OTP, users, user_sync |
| `../WORKFLOW_PUSH.md` | Expo push, FCM, Cloud Function, chat notifications |

## General playbooks (`workflows/`)

| Playbook | When |
|----------|------|
| `new-feature.md` | New user-facing capability |
| `bugfix.md` | Regression or broken behavior |
| `firebase-change.md` | Firestore, Auth, Storage, indexes, rules |
| `mobile-change.md` | Expo app only (`mobile/`) |
