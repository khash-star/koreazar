# Incidents

Production or high-severity issues. One file per incident from `INCIDENT_TEMPLATE.md`.

## Naming

`YYYY-MM-DD-short-title.md`

## When to create

- Production outage or broken core flow (auth, listings, payments path)  
- Bad deploy or Firebase rules/index failure in production  
- Data loss or wrong Firebase project  

## Link updates

After closing an incident, run `../memory-updates/bug-memory-update.md` and update `../summaries/known_bugs.md`.
