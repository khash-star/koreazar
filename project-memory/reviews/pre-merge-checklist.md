# Pre-merge Checklist

## Scope & safety

- [ ] Diff is minimal and matches the request  
- [ ] No `node_modules/`, `dist/`, vendor, or autogenerated edits  
- [ ] No deployment config edits unless explicitly requested  

## Architecture

- [ ] Web changes in repo root `src/`; mobile in `mobile/` only  
- [ ] No duplicate services or API wrappers  
- [ ] Data/query logic not merged into pure UI components  
- [ ] Agents SQL (or service layer) separate from display layer  

## Product-critical

- [ ] Admin routes and `users.role` checks intact  
- [ ] Session / auth persistence unchanged (unless task was auth)  
- [ ] Firebase Auth, Firestore, Storage paths consistent  
- [ ] FCM / push paths unchanged (unless task was notifications)  

## Firebase / deploy (if touched)

- [ ] `firestore.indexes.json` matches new queries  
- [ ] Rules documented in `SECURITY.md` spirit  
- [ ] `npm run sync-listings` run if web listing constants changed  

## Verification

- [ ] Build or dev command identified  
- [ ] Manual test steps documented  
- [ ] Change report complete (files, why, risks, test)
